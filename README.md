# Testcase Review Agent

基于 ISO 25010 质量模型的 AI 九方评审测试用例智能体。输入产品需求文档（PRD），自动生成完整的、可评审的、可直接执行的测试用例交付物。

## 功能特性

**五阶段自动化流水线：**

| 阶段 | 说明 | 输出 |
|------|------|------|
| Stage 0 | 需求抽取与规范化 | 模块功能点、接口清单、状态转换、需求疑点、RTM |
| Stage 1 | 原始测试用例集生成 | 覆盖正向/异常/边界/权限场景，P0-P3 优先级 |
| Stage 2 | AI 九方评审（ISO 25010） | 9 个角色维度的评审意见和缺陷发现 |
| Stage 3 | 用例优化 | 根据评审意见补充、修正、新增用例 |
| Stage 4 | 报告生成 | 评审评分、覆盖度分析、HTML 报告 + Excel 导出 |

**九方评审角色：**

| 角色 | 职责 | ISO 25010 质量特性 |
|------|------|-------------------|
| Arch 开发架构师 | 模块性、可重用性、可修改性 | 可维护性、可移植性 |
| Dev 开发工程师 | 功能正确性、边界处理、异常分支 | 功能适合性 |
| QA 测试工程师 | 场景覆盖、回归风险、容错性 | 可靠性、可测试性 |
| PM 产品经理 | 功能完整性、业务规则、用户旅程 | 功能适合性 |
| UX 用户体验设计师 | 易操作性、用户差错防御、界面美感 | 易用性 |
| Sec 安全工程师 | 保密性、完整性、可审计性 | 安全性 |
| Ops 运维工程师 | 监控告警、灰度回滚、易恢复性 | 可维护性、可靠性 |
| Perf 性能工程师 | 响应时间、并发承载、资源利用 | 性能效率 |
| DBA 数据工程师 | 数据一致性、迁移兼容性、备份恢复 | 数据完整性 |

## 快速开始

### 前置条件

- [Claude Code](https://claude.ai/code) CLI 或桌面应用

### 使用方式

1. 克隆仓库：

```bash
git clone https://github.com/MaoDou-Yan/testcase-review-agent.git
cd testcase-review-agent
```

2. 在 Claude Code 中打开项目目录，将 PRD 文档放入 `input/` 目录：

```
input/
  你的需求文档（PRD）.md
```

3. 在 Claude Code 中执行：

```
根据 input 中的prd，来执行这个agent
```

4. Agent 将自动完成五阶段流水线，生成 `output/report.html` 报告文件。

5. 在浏览器中打开 `output/report.html`，可查看完整报告并导出 Excel。

### 支持的输入格式

- Markdown (.md)
- Word/PDF 文档
- 产品原型截图
- 纯文本需求描述
- 表格数据

### 输出物

所有输出文件统一存放在 `output/` 目录：

| 文件 | 说明 |
|------|------|
| `output/report.html` | 完整交互式 HTML 报告（含 5 个 Tab） |
| `output/*.xlsx` | 通过报告内「下载Excel」按钮生成，含 3 个 Sheet |

## 报告功能

HTML 报告包含 5 个标签页：

1. **需求抽取清单** — 模块功能点、接口清单、状态转换、需求疑点（含风险等级标签）、状态机覆盖
2. **原始用例集** — 完整测试用例表，支持模块/优先级筛选和关键词搜索
3. **AI 评审详情** — 九方评审发现，支持按角色/缺陷类型筛选
4. **优化后用例集** — 优化后的完整用例集，含优化记录
5. **评审与覆盖度报告** — 九方评分卡、问题分布、质量雷达、RTM 追踪矩阵

**交互功能：**
- 在线编辑：开启编辑模式后可直接修改单元格内容
- 新增/删除行：支持在报告内新增和删除用例行
- 撤销全部：一键恢复到初始状态
- 下载 Excel：基于当前页面数据（含编辑）实时生成 .xlsx 文件
- 复制数据：复制当前筛选后的优化用例到剪贴板

## 用例编号规范

用例采用 `PrimaryModule-SecondaryModule-NNN` 格式：

| 前缀 | 模块 | 示例 |
|------|------|------|
| QA | 用户端问答 | QA-Send-001, QA-Match-003 |
| Session | 会话记录 | Session-History-002 |
| KB | 词条管理 | KB-Add-005, KB-Batch-002 |
| Unknown | 未知问题 | Unknown-Collect-001 |
| Stats | 数据统计 | Stats-Basic-003 |
| Perm | 权限设计 | Perm-001 |
| Exc | 异常场景 | Exc-Net-001 |
| Perf | 性能需求 | Perf-001 |
| Compat | 兼容性 | Compat-001 |
| Security | 安全需求 | Security-001 |
| Maint | 可维护性 | Maint-001 |

## 优先级定义

| 等级 | 定义 | 典型场景 |
|------|------|---------|
| P0 | 主链路阻断、结算/库存/凭证影响、不可逆数据影响 | 核心流程正向验证、数据安全 |
| P1 | 核心异常流、接口失败、幂等性、并发、数据一致性 | 异常处理、权限校验、状态转换 |
| P2 | 边界值、审计日志、重试细节、UI 交互、兼容性 | 边界测试、浏览器兼容、日志验证 |
| P3 | 低风险展示、辅助提示、美化类、非阻断校验 | 文案校验、样式检查 |

## 评分规则

每个评审角色起始分 100 分，根据发现的缺陷类型扣分：

| 缺陷严重度 | 每个扣分 |
|-----------|---------|
| P0 用例的逻辑遗漏/场景遗漏 | -8 |
| P0/P1 用例的安全漏洞/数据一致性风险 | -6 |
| P0/P1 用例的性能风险/数据完整性风险 | -6 |
| P0/P1 用例的边界遗漏 | -6 |
| 任何用例的可验证性不足 | -4 |
| 任何用例的运维盲区/架构腐化风险 | -4 |
| 任何用例的表述不清/优先级不准 | -2 |
| 任何用例的易用性缺陷 | -2 |
| UX 角色的交互不符 | -3 |
| 无功能影响的措辞建议 | -1 |
